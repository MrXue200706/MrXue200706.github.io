移动web

移动端布局的三种方式：

百分比布局

页面中随视口大小改变的元素使用百分比表示；不随视口大小改变的元素使用固定宽度
二倍图：由于不同手机像素密度（PPI）不同，同样大小的图片，在高PPI的手机上会缩小，为了显示正常大小，要放大，但是会失真，所以要提供二倍大小的图片供放大而不失真
两个重要的css库：1.reset.css清除浏览器默认样式；2.normalize.css解决不同浏览器的css兼容问题

rem布局

rem是相对root根元素大小的布局，通常是根元素的字体大小

视口
  • 视口：layout viewport网页布局视口；ideal viewport设备视口大小
  • 视口设置：

    initial-scale初始缩放比；maximum-scale允许用户的最大缩放比；minimum-scale最小缩放比；user-scalable是否允许用户缩放，yes/no

媒体查询

媒体查询是CSS3的中一个强大的工具。它是包含媒体类型以及零或多个检查特定媒体特征条件的表达式。表达式描述了媒体特征, 最终会被解析为 true 或 false.如果是true 那么会执行 如果是false 就不会执行了.通过不同的判别条件识别不同的设备 以给该设备应用不同的样式

  2. style里面引入：
  1. response.js解决媒体查询的兼容性问题，使得不支持媒体查询的浏览器也可以使用媒体查询
布局
  • 布局时，640px大小的设计图，元素大小为:实际大小/32rem

    因为把宽为640px的设计图分为20份，每份大小就是1rem=640/20=32px,，1rem元素在不同大小的屏幕上也应该是1rem大小，实际元素在640px屏幕中所占的大小用rem表示，即为几个rem，实际大小/32rem,则在不同的屏幕上也应该占：实际大小/32rem；其他尺寸的设计图以此类推

响应式布局

利用媒体查询，在不同大小的屏幕下，显示不同的样式

屏幕大小的划分
  1. XS：小于768px，手机 @media(max-width:@screen-xs-max){}
  2. sm:大于等于768px，小于992px ，平板 @media(max-width:@screen-sm-max)and(min-width:@screen-sm-min){}
  3. md：大于等于992px，小于1200px，桌面显示器 @media(max-width:@screen-md-max)and(min-width:@screen-md-min){}
  4. lg：大于等于1200px，大屏幕 @media(min-width:@screen-lg-min){}
